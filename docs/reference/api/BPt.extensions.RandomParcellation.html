class BPt.extensions.RandomParcellation(geo, n_parcels, medial_wall_inds=None, medial_wall_mask=None, random_state=None)[source]Â¶

This object can be used to generate a random surface parcellation.

Parameters
geolist of array-like

geo, or geometry, should be passed as a list of arrays (or sets, or lists), where each index of the outer list is associated with that vertex index. Each list should contain all of that vertexâs valid neighbors. So for example, if the vertex at index 0 has neighbors 5, 8, 9, 10, then the first element of passed geo should be [5,8,9,10].

n_parcelsint

The number of random parcels to generate. Will generate as 1-n_parcels, with 0 reserved for non-parcels.

medial_wall_indsarray-like of int or None, optional

By passing an array-like containing valid integers. between 0 and the size of the parcel, this parameter can specify optionally index where parcels should not be generated.

This parameter should be left as None if passing medial_wall_mask, as only one will be used.

default = None
medial_wall_maskarray-like of bool or None, optional

Simmilar to medial_wall_inds, you may also specify the medial wall, or spots not to fill, via this parameter by passing a boolean mask, where 1 or True signifies that vertex is part of the medial wall to exclude.

This parameter should be left as None if passing medial_wall_inds, as only one will be used.

default = None
random_stateint, None, or RandomState, optional

A random seed to be passed to create a numpy RandomState, as either a numpy random state, an integer seed, or None to generate a random RandomState.

default = None
